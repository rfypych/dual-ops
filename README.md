# 🌌 Dual-Ops: Autonomous AI Bug Bounty Bridge

**Dual-Ops** is a lightweight, high-performance bridge system designed to connect external AI agents (like z.ai and Qwen Coder) directly to a local Kali Linux/WSL terminal. 

It enables a **Commander-Oracle** autonomous workflow where one AI (the Commander) manages field operations while another (the Oracle) develops custom exploits and analyzes complex data—all synchronized through an isolated local workspace.

## 🚀 The Dual-Ops Architecture

The system is designed with a clean split between infrastructure and operations:

1.  **Dual-Ops Bridge (This Repo):** The core communication infrastructure. Exposes your terminal securely via Cloudflare Tunnels.
2.  **Bounty Hunter Repo (Your Workspace):** A separate, private repository where all scans, targets, exploits, and agent communication logs are stored. This keeps your 'Dual-Ops' code clean for publication while keeping your sensitive data private.

## ✨ Features

- **Autonomous Workflow** - Predesigned prompt loops for multi-agent collaboration.
- **Shared Queue System** - File-based communication bus for inter-agent task delegation.
- **Secure Tunneling** - Public access to local WSL/Kali via Cloudflare without port forwarding.
- **Binary-Safe Transfer** - Upload and download tools, scans, and exploits in real-time.
- **Stealth Identifying** - Automatic X-HackerOne header injection for bug bounty compliance.

## 🛠 Quick Start

### 1. Initialize the Bridge
Run the launcher to start the agent and establish the tunnel:
```powershell
python launch.py
```
Note the **Public URL** generated by Cloudflare.

### 2. Setup the Agents
Use the provided [ai_prompts.md](./ai_prompts.md) to initialize your AI agents:
- **z.ai (Commander):** Paste the Commander prompt and insert your Bridge URL.
- **Qwen Coder (Oracle):** Paste the Oracle prompt and insert your Bridge URL.

### 3. Deploy
Start the operation by giving z.ai a target:
> "Target identified: `example.com`. Begin full autonomous reconnaissance."

## 📖 Directory Structure

- `bridge_agent.py`: High-performance REST server for terminal access.
- `bridge_controller.py`: Client-side tool for manual interaction.
- `launch.py`: Automated setup for agent + tunnel.
- `ai_prompts.md`: The "brains" - system prompts for your AI agents.

## 🔐 Security & Compliance

- **X-HackerOne Headers**: Ensure all agents are configured with the mandatory `X-HackerOne: [username]` header.
- **API Authentication**: Enable optional Bearer token protection in `bridge_agent.py`.
- **Isolated Workspace**: Always keep your `bounty_hunter_repo` separate from your public `dual-ops` code.

## 📋 Requirements

- Python 3.7+
- `requests` (for controller)
- `cloudflared` (installed in PATH)
